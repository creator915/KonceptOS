# 坦克大战 (FC复古版) 需求文档

> 8-bit像素风格 | NES/Famicom平台 | 1988年风格

---

## 1. 游戏概述

### 1.1 产品简介
一款致敬经典的8-bit像素风格坦克对战游戏。在复古的8-bit画面中驾驶坦克，与敌人展开激烈的像素坦克大战。简单上手，紧张刺激！

### 1.2 游戏类型
- 横版射击游戏
- 单机/双人对战

### 1.3 目标用户
- 怀旧游戏玩家
- 8-bit/FC游戏爱好者
- 休闲益智玩家

### 1.4 核心玩法
- 操控坦克摧毁所有敌人
- 击毁敌方坦克获得分数
- 躲避敌人子弹和障碍物
- 每关结束进入Bonus关卡

## 2. 功能模块

### 2.1 战斗系统
| 模块 | 功能描述 |
|------|----------|
| 移动控制 | 上下左右移动，8方向 |
| 射击系统 | 单发/连发子弹，间隔0.3秒 |
| 爆炸效果 | 像素爆炸动画，3帧 |
| 敌人AI | 3种敌人巡逻/追击/固定 |
| 关卡设计 | 5大地图，每关3阶段 |

#### 性能影响
- **射击频率**：0.3秒间隔限制子弹密度，高峰同时屏幕5-8发子弹，碰撞检测为 O(子弹×目标数)
- **爆炸动画**：3帧短动画，利用精灵复用减少纹理切换开销；爆炸半径碰撞检测需遍历瓦片
- **8方向移动**：玩家输入每帧采样（60fps），方向切换响应时间 <16ms，无输入缓冲延迟

### 2.2 坦克系统
| 模块 | 功能描述 |
|------|----------|
| 我方坦克 | 像素坦克，4方向 sprite |
| 敌方坦克 | 3种类型：轻型/中型/重型 |
| 坦克强化 | 击毁特定敌人获得强化道具 |
| 坦克伤害 | 1-3发子弹击杀 |

#### 性能影响
- **精灵渲染**：每辆坦克 4方向×2帧行走动画，最多同屏6-8辆坦克，合计 <64 个精灵图块，VRAM 压力极小
- **碰撞箱**：每坦克一个 AABB 碰撞框，碰撞检测 O(n²) 对手数量，8辆坦克约 28 次检测/帧，开销可忽略
- **火力强化**：3级火力影响子弹数量（1/2/3发），每增一级增加一帧碰撞检测目标，轻重可控

### 2.3 道具系统
| 模块 | 功能描述 |
|------|----------|
| 炸弹 | 清屏所有敌人 |
| 盾牌 | 无敌5秒 |
| 星星 | 火力升级（最多3级） |
| 坦克 | 额外一条命 |
| 降落伞 | 减速敌人 |

#### 性能影响
- **炸弹清屏**：触发时需同时销毁所有敌人精灵并播放爆炸序列，敌人越多帧开销越大；建议最多 40 敌人同屏，爆炸动画分 3 帧交错执行以平滑帧率
- **盾牌无敌**：5秒无敌状态需每帧额外绘制防护罩光晕效果（每帧画一个半透明圆环），对帧时间影响 <5%
- **减速效果**：降低敌人速度变量，影响所有敌人 AI 更新逻辑，减速开启时每帧额外一次全局速度乘法，开销极小

### 2.4 关卡系统
共20关，分为2大章节，每章10关，最后一关为章节Boss战。

#### 第一章：训练战役 (关卡 1-10)

|------|------|----------|------|

| 2 | 草地 | 12 | 简单敌人 |
| 3 | 草地 | 14 | 移动加速 |
| 4 | 树林 | 16 | 隐蔽地形 |
| 5 | 树林 | 18 | 巡逻敌人 |
| 6 | 河流 | 18 | 水域障碍 |
| 7 | 河流 | 20 | 混合地形 |
| 8 | 沙漠 | 20 | 快速敌人 |
| 9 | 沙漠 | 22 | 追击敌人 |
| 10 | 沙漠 | 25 | **章节Boss：沙虫** |

#### 第二章：最终决战 (关卡 11-20)
| 关卡 | 主题 | 敌人数量 | 特点 |
|------|------|----------|------|
| 11 | 城市 | 22 | 巷战地形 |
| 12 | 城市 | 24 | 固定炮台 |
| 13 | 废墟 | 26 | 混合敌人 |
| 14 | 废墟 | 28 | 障碍密集 |

#### Boss一览
| Boss | 出现关卡 | 血量 | 攻击方式 |
|------|----------|------|----------|
| 沙虫 | 10 | 高 | 钻地突袭、扇形攻击 |
| 超级坦克 | 20 | 极高 | 全范围轰炸、护盾 |

#### 性能影响
- **瓦片渲染**：每帧需重绘 30x15=450 格地砖，砖墙破碎时触发局部瓦片更新；水面/森林需透明度混合，增加填充率开销
- **碰撞检测**：瓦片级碰撞查询，敌人子弹与坦克/地形逐帧检测，最坏情况 O(子弹数×瓦片数)
- **敌人数量**：随关卡递增（10→40），每增加一个敌人增加一帧内 AI 计算和碰撞检测开销，需控制在 60fps 内
- **Boss 战**：沙虫钻地需切换两套渲染状态，超级坦克护盾需额外绘制抗锯齿圆弧，帧时间预算更紧张

### 2.5 分数系统
| 项目 | 分数 |
|------|------|
| 轻型坦克 | 100 |
| 中型坦克 | 200 |
| 重型坦克 | 300 |
| Boss | 5000 |
| 强化道具 | 50 |
| 存活奖励 | 1000/关 |

#### 性能影响
- **分数计算**：每次击杀/拾取触发一次整数累加和 HUD 重绘；HUD 更新仅重绘数字区域（4-6位），无需刷新全屏，开销极小
- **最高分持久化**：每关结束后写入 2KB SRAM，电池供电确保断电后数据保留；写入为阻塞操作，约 1-2ms，需避免在游戏循环中同步写入

### 2.6 敌人AI系统
| 类型 | 行为描述 | 代码开销 |
|------|----------|----------|
| 巡逻型 | 固定路线往返移动 | 每帧位置插值，无寻路 |
| 追击型 | 检测到玩家后直线追击 | 每帧距离计算 + 方向判断 |
| 固定/炮台式 | 原地射击，不移动 | 每帧子弹生成，无移动计算 |

#### 性能影响
- **寻路开销**：所有敌人使用简单直线追击，无 A* 寻路；追击型敌人每帧计算一次欧几里得距离（开方运算可优化为距离平方比较），开销极小
- **同屏AI上限**：建议同时激活 ≤40 个敌人AI个体，每帧额外 CPU 开销 <3%，轻松满足 60fps
- **难度缩放**：随关卡编号增加敌人AI反应速度和射击频率，参数调整在配置表而非代码中，不影响运行时性能

### 3.1 游戏模式
| 模式 | 描述 | 玩家数 |
|------|------|--------|
| 单人模式 | 闯关打Boss | 1人 |
| 双人合作 | 双人同屏闯关 | 2人 |
| 双人对战 | 分屏对战 | 2人 |

#### 性能影响
- **同屏双人合作**：同时渲染两辆玩家坦克 + 双倍敌人数量，每帧碰撞检测翻倍（约 56 次 AABB 检测），需确保 60fps
- **分屏对战**：屏幕纵向二分，每个视口独立渲染 30x15 瓦片地图，渲染工作量约单人的 2.1 倍（视口裁剪边界额外计算）
- **存档/读档**：合作通关后一次性写入进度数据到 SRAM，与游戏循环无冲突

### 3.2 操作方式
| 操作 | 玩家1 | 玩家2 |
|------|-------|-------|
| 移动 | 方向键 | WASD |
| 射击 | 空格/Z | X/Enter |
| 暂停 | P | P |

#### 性能影响
- **手柄/键盘轮询**：每帧（60fps）轮询一次手柄状态，键盘使用系统中断事件驱动，两路输入合并后按位取或，无额外性能负担
- **双人冲突按键**：禁止同一帧内同时触发双方射击，避免双份子弹导致子弹池溢出；输入缓冲队列深度建议 2

|------|----------|----------|
| 玩家1 | 3条 | 5条 |
| 玩家2 | 3条 | 5条 |

## 4. 系统架构

### 4.1 客户端架构
```
┌─────────────────────────────────┐
│         游戏主循环               │
│     (60fps / 50fps兼容)         │
├──────────┬──────────┬──────────┤
│ 渲染模块  │ 输入模块  │ 音频模块  │
│ 16x16    │ 手柄/键盘 │ 8-bit    │
│ 像素块    │ 支持     │ 声道     │
├──────────┴──────────┴──────────┤
│         游戏逻辑层               │
│   碰撞检测 / AI / 状态机        │
├─────────────────────────────────┤
│         地图数据层               │
│     Tile Map (30x15格)          │
└─────────────────────────────────┘
```

### 4.2 技术规格
| 项目 | 规格 |
|------|------|
| 分辨率 | 256x240 (NES标准) |
| 色彩数 | 最多54色同屏 |
| 精灵大小 | 8x8 / 16x16 / 32x32 |
| 地图大小 | 30x15 格 |
| 帧率 | 60fps (NTSC) / 50fps (PAL) |
| 音频 | 2个方波 + 1个三角波 + 1个噪声 |

#### 性能影响
- **帧率稳定性**：60fps 意味着每帧 ≤16.67ms；渲染瓦片地图占 6-8ms，碰撞检测占 1-2ms，AI 计算占 2-3ms，音频混音占 <1ms，留有充足余量
- **同屏精灵上限**：建议不超过 64 个活动精灵（含子弹、爆炸、道具），超过后需启用硬件精灵数量限制或分层裁剪
- **色彩限制**：54 色同屏约束调色板设计，避免同帧切换调色板造成额外 DMA 开销

### 4.3 数据存储
| 类型 | 容量 | 内容 |
|------|------|------|
| SRAM | 2KB | 最高分、进度 |
| 卡带 | 可选电池存档 | 继续游戏 |

#### 性能影响
- **SRAM 写入**：游戏循环外触发（关卡切换/暂停菜单），写入 2KB 数据约 1-2ms，同步阻塞但不冻结游戏画面（写入期间游戏已暂停）
- **SRAM 读取**：启动时加载最高分和关卡进度，读取时间 <5ms，不影响游戏启动体验
- **电池存档**：断电保护确保存档不丢失，无需额外校验算法，降低 CPU 负载

### 4.4 音效设计
| 音效 | 类型 | 用途 |
|------|------|------|
| BGM | 4声道循环 | 背景音乐 |
| 射击 | 方波短音 | 开火音效 |
| 爆炸 | 噪声+方波 | 爆炸音效 |
| 获得道具 | 上升音阶 | 吃道具音效 |
| 死亡 | 下降音阶 | 坦克爆炸 |

#### 性能影响
- **4声道混音**：每帧需混合 4 路音频样本，NES APU 硬件完成 CPU 零开销；软件层面仅负责触发音符事件，开销极小
- **BGM 循环**：单帧仅需检查曲目循环标志位，不影响渲染帧时间
- **爆炸音效**：噪声通道与方波通道同时发声，短暂占用 2/4 声道；爆炸期间射击音效被压低，音频优先级逻辑简单
