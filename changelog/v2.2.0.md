# Changelog v2.2.0

## Release Date
2026-04-01

## Feature: Build + Selenium 自动验证循环

### 概述

本次更新为 `konceptos build` 命令添加了 Selenium 自动测试和 LLM 引导的重新生成循环。

### 新增功能

#### 1. SeleniumTestRunner (`konceptos/test_runner.py`)
- 新增 `SeleniumTestRunner` 类，支持自动化浏览器测试
- 支持测试钩子加载（`TEST_INPUTS`, `TEST_OUTPUTS`, `CANVAS_CHECK_JS`）
- 包含超时保护和线程隔离，避免 Selenium 挂起
- 无 Selenium 环境时自动 fallback 到基础验证

#### 2. LLM 新增方法 (`konceptos/llm.py`)
- `build_full_with_hooks()`: 生成 HTML 同时生成测试钩子
- `should_regenerate()`: LLM 判断测试失败是否值得重新生成
- `analyze_errors()`: LLM 分析错误并给出修正建议

#### 3. Engine 新增方法 (`konceptos/engine.py`)
- `interface_boundaries()`: 从 G/M/I 提取接口边界信息
  - inputs: W/RW 权限的属性（外部输入）
  - outputs: R/RW 权限的属性（预期输出）

#### 4. Build 命令扩展 (`konceptos/cli.py`)
```bash
konceptos build output.html --auto-test --max-iter 3 --test-hooks selenium_test_hooks.py
```
- `--auto-test`: 是否运行 Selenium 测试（默认开启）
- `--max-iter`: 最大重新生成次数（默认 3）
- `--test-hooks`: 测试钩子文件名

#### 5. 依赖更新 (`pyproject.toml`)
- 新增 `selenium`
- 新增 `webdriver-manager`

#### 6. 文档 (`COMMANDS.md`)
- 新增 Build 命令说明
- 新增工作流示例

### 技术细节

#### 接口边界提取
```
Engine.interface_boundaries() 返回:
{
  'inputs': [
    {'attr': 'A01', 'name': 'deck_cards', 'desc': '...', 'writers': ['deck_manager'], 'schema': '...'},
    ...
  ],
  'outputs': [
    {'attr': 'A02', 'name': 'player_hand', 'desc': '...', 'readers': ['ai_engine'], 'schema': '...'},
    ...
  ],
  'schemas': {...}

```

#### 测试钩子格式
```python
TEST_INPUTS = [
    ("bid_button", "#btn-bid-1", "click"),
    ("play_button", "#btn-play", "click"),
    ...
]

    ("game_canvas", "#gameCanvas", "canvas_has_content"),
    ("status_bar", "#status-bar", "exists"),
    ...
]

CANVAS_CHECK_JS = """
(function() {
    var canvas = document.getElementById('gameCanvas');
    // ... pixel sampling to verify canvas has content
})();
"""
```

## 测试报告

### 测试项目：斗地主（Dou Di Zhu）游戏

**测试文件**: `tests/prd.md`

**测试命令**:
```bash
cd tests
konceptos llm analyze prd.md
konceptos evolve 5
konceptos schema --auto
konceptos build output.html --auto-test
```

### 测试结果

| 阶段 | 状态 | 详情 |
|------|------|------|
| LLM 分析 | ✅ 通过 | 提取 8 个对象、12 个属性 |
| RW 拆分 | ✅ 通过 | 经过 5 轮 evolve，拆分为 26 个对象 |
| Schema 生成 | ✅ 通过 | 12 个属性全部生成 TypeScript 类型 |
| Build 生成 | ✅ 通过 | 生成 40KB HTML+Canvas 页面 |
| 测试钩子生成 | ✅ 通过 | 生成完整的 selenium_test_hooks.py |
| 基础验证 | ✅ 通过 | HTML 包含 canvas 元素 |

### 生成产物

| 文件 | 大小 | 说明 |
|------|------|------|
| `output.html` | 40,643 bytes | 斗地主游戏完整 HTML+JS+Canvas |
| `selenium_test_hooks.py` | 5,502 bytes | Selenium 测试钩子 |

### 测试钩子覆盖

**输入测试** (7 项):
- 叫地主按钮（1/2/3 点）
- 不叫按钮
- 出牌按钮
- 不出按钮
- 提示按钮
- 重新开始按钮
- Canvas 点击

**输出测试** (12 项):
- Canvas 内容检查（像素采样）
- 状态栏存在性和文本检查
- 所有游戏按钮存在性检查
- 结果对话框检查

**Canvas 状态检查**:
- 像素采样验证（每 100 像素采样）
- 检测非背景色像素数量 > 50 视为有内容
